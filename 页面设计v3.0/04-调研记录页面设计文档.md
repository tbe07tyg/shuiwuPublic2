# 📊 调研记录页面设计文档 v2.0

## 📄 页面基础信息

- **页面名称**: 调研记录
- **路由地址**: `/research/results`
- **组件路径**: `src/views/research/results/index.vue`
- **设计版本**: v2.0
- **更新时间**: 2025-01-17

## 🔄 v2.0版本变更说明

### 📝 菜单名称变更
- **v1.0**: 调研成果
- **v2.0**: 调研记录

### 🎯 设计理念
- **一目了然**: 调研过程和结果在同一页面清晰展示
- **时间线展示**: 以时间线形式展现调研全过程
- **结果导向**: 突出调研结论和后续行动
- **简化操作**: 减少页面跳转，提高查看效率
- **便捷填写**: 简单明了的调研事项添加，随时可添加新事项

## 🎯 简化填写体验设计

### 📝 核心设计理念

#### 1. **最少输入原则**
- ✅ 只要求填写必需信息
- ✅ 提供智能默认值
- ✅ 支持快速选择选项
- ✅ 减少文字输入量

#### 2. **随时可添加**
- 🚀 每个调研卡片都有"添加事项"按钮
- 🚀 支持内联编辑，无需跳转页面
- 🚀 快速弹窗，500px小窗口
- 🚀 移动端支持滑动快速操作

#### 3. **智能辅助填写**
- 🤖 自动填充负责人（当前用户）
- 🤖 智能推荐事项类型
- 🤖 自动计算预计完成时间
- 🤖 历史事项模板快速复用

### 📋 填写字段最小化

#### 必填字段（只有1个）
- **事项内容** *：简要描述即可，200字符限制

#### 可选字段（有智能默认值）
- **计划时间**：默认"今天"
- **预计用时**：默认"1天"  
- **负责人**：默认当前用户
- **事项类型**：默认"其他"
- **关联文档**：可选上传

### 🚀 快速操作设计

#### 1. **一键添加**
```
[事项内容____________] [今天▼] [1天▼] [✓添加]
```

#### 2. **批量模板**
- 预设常用调研事项模板
- 一键添加标准调研流程
- 自定义保存个人模板

#### 3. **快速更新**
- 进度滑块直接拖拽更新
- 状态一键切换
- 里程碑快速设置

### 📱 移动端优化

#### 1. **大按钮设计**
- 添加事项按钮高度48px
- 操作按钮间距16px
- 输入框高度44px

#### 2. **滑动操作**
- 左滑快速添加事项
- 右滑快速更新进度  
- 长按快速操作菜单

#### 3. **语音输入**
- 支持语音转文字
- 快速记录调研进展
- 智能识别关键信息

## 📊 数据结构优化

### 调研事项数据模型（简化版）
```javascript
{
  id: 'T2024001',
  researchId: 'R2024001', // 关联调研记录
  content: '安排供应商技术交流会议', // 事项内容（必填）
  type: 'technical', // 事项类型
  planDate: '2024-01-25', // 计划时间
  estimatedHours: 4, // 预计用时（小时）
  responsible: '张三', // 负责人
  status: 'pending', // pending, active, completed
  progress: 0, // 完成进度 0-100
  actualStartDate: null, // 实际开始时间
  actualEndDate: null, // 实际完成时间
  notes: '', // 备注说明
  attachments: [], // 关联文档
  createdAt: '2024-01-24T10:30:00Z',
  updatedAt: '2024-01-24T10:30:00Z'
}
```

### 快速操作API设计
```javascript
// 快速添加事项
POST /api/research/items/quick
{
  content: "事项内容",
  researchId: "R2024001",
  planDate: "2024-01-25", // 可选，默认今天
  estimatedHours: 4 // 可选，默认8小时
}

// 快速更新进度
PATCH /api/research/items/{id}/progress
{
  progress: 80,
```
